Description

소프트웨어가 발전하고 복잡도가 상승함에 따라 회사에 도입된 전체 IT 시스템을 효율적으로 관리하는 것은 점점 더 어려워지고 있습니다. 서로 분리된 복잡한 영역별 시스템들은 종류가 다른 로그 데이터를 계속해서 생성하고 있습니다. 생성되는 로그 데이터를 수집하여 가공하면 회사의 여러 문제 즉시 해결에 활용할 수 있는 정보가 됩니다. 이렇게 회사 전반의 데이터를 수집하여 필요할 때 회사의 의사결정에 즉시 활용하는 분야를 Operational Intelligence(OI)라고 합니다. 하지만 이런 기술을 적용하는 것은 쉬운 일이 아닌데, 많은 IT 시스템이 로그 데이터의 통합에 대해 고려하지 않고 있어 통합 작업에 시간과 비용이 많이 들기 때문입니다. Herring은 OI를 수행하기 위해 서로 다른 종류의 실시간 로그 데이터를 효과적으로 수집하여 관리하고 필요할 때 데이터를 즉시 열람할 수 있는 오픈 소스 프로젝트 입니다.

현재 동작중인 시스템에 Herring을 적용하면, 시스템에서 생성되는 다양한 형태의 데이터를 실시간으로 볼 수 있습니다. 이러한 정보를 이용해 다음과 같은 경우에 활용 할 수 있습니다.

  • 동작중인 장비에 대한 실시간 상태 모니터링
  • 현재 진행중인 서비스에 대한 고객의 수요 형태와 경향 분석을 이용한 수익 증대
  • 서비스 이용 고객들의 접근 로그 분석을 통한 비정상적인 접근 감지
  • 시스템에서 발생한 오류를 실시간으로 대응

Roles

  • Herring System을 전반적으로 다룰 수 있는 Web-based Herring Manager를 담당하였습니다.
    • Spring Framework 3.2 기반으로 구현하였습니다.
  • 쉽게 자료를 검색할 수 있는 Herring Query Language를 설계하였습니다.
    • ANTLR 라이브러리를 통해 Query Language를 직접 설계하였습니다.
    • 입력된 Query에 대해서 실제 검색을 수행하기 위한 Query Planning 모듈도 설계하였습니다.
